1.Easy tasks with low level of noise¶

At the very begining, let's try to solve a very simple problem. Which is to use LSTM learn the transformation from sin(x) to cos(x). This question exactly has no noise at all. So we don't need to think about the overfitting problem. Let's just write a write LSTM structure to deal with it.

INPUT_SIZE = 1      # rnn input size
CELL_SIZE = 64      # rnn cell size
LR = 0.02           # learning rate

tf.reset_default_graph()
tf_x = tf.placeholder(tf.float32, [None, TIME_STEP, INPUT_SIZE])        # shape(batch, 5, 1)
tf_y = tf.placeholder(tf.float32, [None, TIME_STEP, INPUT_SIZE])          # input y

lstm_cell = tf.contrib.rnn.BasicLSTMCell(num_units=CELL_SIZE)
init_s = lstm_cell.zero_state(batch_size=1, dtype=tf.float32)    # very first hidden state
outputs, final_s = tf.nn.dynamic_rnn(
    lstm_cell,                   # cell you have chosen
    tf_x,                       # input
    initial_state=init_s,       # the initial hidden state
    time_major=False,           # False: (batch, time step, input); True: (time step, batch, input)
)
outs2D = tf.reshape(outputs, [-1, CELL_SIZE])                       # reshape 3D output to 2D for fully connected layer
net_outs2D = tf.layers.dense(outs2D, INPUT_SIZE)
outs = tf.reshape(net_outs2D, [-1, TIME_STEP, INPUT_SIZE])          # reshape back to 3D

loss = tf.losses.mean_squared_error(labels=tf_y, predictions=outs)  # compute cost
train_op = tf.train.AdamOptimizer(LR).minimize(loss)

sess = tf.Session()
sess.run(tf.global_variables_initializer())     # initialize var in graph

It seems that the latter one can provide more information and it's a bargin. However, for LSTM and RNN. Sometimes, we should rely on the fitting on sequence to sequence. 10 points to 10 points may provide 'more information' for the learner to learn this method. The learner will not suck in some fake equlibrium.

What is fake equlibrium? Sometimes, you think you get a good regression results. Because the real value and predicted value are so close that you can't recognize the difference. If you just let t-1's value to predict t's value. It could also get 'good performance'. Here, the leaner sucks in the fake equlibrium, it doesn't learn the tendency but just repeat the past value. Maybe it just do a EMA on the past value not only just SMA.

In this task, learn the transformation from sin(x) to cos(x) is too easy to be solved. You can try to use DNN to do this task. And I think it can perform well both on 10 to 1 and 10 to 10.

What I want to say is that, people may easily think that they have known LSTM's programming. However, in this simple case, no matter what you did, you will receive a good consequence. But whether you use dropout, drop input or output, use what kind of method to do normalization. All these kind of settings will show big differences when you deal with difficulty tasks. Later on, we will dig out more in the highly difficulty questions: Stock Time Series. And I will show you the tricks and differences.

2.1 The normalization method is important¶

Normally, we will do mean-std normalization new_x=(x-mean)/std. It's most frequently used.

In [0]:
def mean_std_normalization_point(raw_data):
    
    #use t to predict t+look
    look=1
    #mean-std normalization, input 10 points predict 1 points
    #eg, in: x1-x10, out: x11, and x11 will be useful.
    train_rate=0.75
    raw_data=pd.DataFrame(raw_data['close'])
    raw_data=raw_data.dropna()
    raw_data=raw_data[raw_data['close']>0]
    raw_data=list(raw_data['close'])

    #get the mean and std
    raw_data=raw_data[:1000]
    basic_sample=raw_data[:int(len(raw_data)*train_rate)]
    avg=np.mean(basic_sample[:int(len(basic_sample)*train_rate)])
    std=np.std(basic_sample[:int(len(basic_sample)*train_rate)])

    #mean-std standardlization
    raw_data=(raw_data-avg)/2/std
    if len(raw_data)%2==0:
        raw_data=raw_data
    else:
        raw_data=raw_data[1:]
    
    data=raw_data
    x=np.full([len(data)-200,22],np.nan)
    for i in range(100,len(data)-100):
        x[i-100,0:20]=np.array(data[i-20:i])
        x[i-100,20]=(data[i-1+look]-data[i-1])/data[i-1]
        x[i-100,21]=data[i-1+look]
        if data[i-1+look]-data[i-1]>=0:
            x[i-100,20]=int(1)
        else:
            x[i-100,20]=int(0)

    point=int(x.shape[0]*train_rate)
    train=x[:point,:].reshape(-1,22)
    test=x[point:,:].reshape(-1,22)
    x_train=train[:,0:20]
    y_train=train[:,20]
    x_test=test[:,0:20]
    y_test=test[:,20]
    lstm_train=train[:,21]
    lstm_test=test[:,21]
    #plt.plot(range(len(x)),x[:,21])
    return x_train,y_train,x_test,y_test,lstm_train,lstm_test

2.2 Dropout is necessary and tricky here¶

And we still have the choice to do dropout. It's hardly to see in most unprofessional article. And I want to stress that if you have not seen a man A silly blog writer use dropout to do LSTM prediction on Stock series, you defintely have no need to read it at all!

2.3 Deceptive performance or predictive performance¶

Deceptive performance means we get a good fitting performance, but it can not tell the tendency, which is useless for equity trading. I (also many researchers and smart blog writers) think the deceptive performance caused by the equilibrium the network can reach. To make some differences in this equilibrium, maybe we can compare the sequence2sequence methods with sequence2point methods.

3.Summary¶

  1. DropoutRet normalization and sequence2sequence is a good match.
  2. Don't be cheated by the fake equilibrium. Although the regression performance is good at a glance, the actual performance should be assessed by comparing with other regression methods. And the fake equilibrium can't tell the tendency. If we don't know tendency, it can't be helpful for our trading
  3. Multi-cells and L1L2 Generalization will also be helpful
  4. Financial problem is difficult, it's non-stationary. You will get different performance in different period of time. You must repeat it many times to measure it. Don't do one experiment and make a conclusion. (The conclusions in this article were made by many trails in my Linux Server. I think it's realiable. But I just test on the latest 3 years A-share Index. Thus, these conclusions are still limited.)
